# Docker & GitHub 急速访问修复器 - 使用说明

## 插件简介

**Docker & GitHub 急速访问修复器** 是一款专门为解决Docker和GitHub访问问题而设计的浏览器插件，具有以下特点：

### 核心功能

1. **Docker访问修复**
   - 自动配置国内镜像加速源
   - 解决Docker Hub访问超时问题
   - 优化镜像拉取速度

2. **GitHub访问修复**
   - 智能hosts文件优化
   - DNS解析加速
   - 镜像源自动切换
   - 资源加载优化

3. **速度优化**
   - 资源缓存机制
   - 连接复用技术
   - 并行加载资源
   - 压缩传输优化

4. **用户体验**
   - 实时访问进度条
   - 平滑动画效果
   - 智能状态检测
   - 直观的设置界面

## 安装步骤

### 方法一：从本地安装

1. **下载插件包**
   - 获取 `docker_github_fixer` 文件夹

2. **打开浏览器扩展管理**
   - Chrome/Edge: 访问 `chrome://extensions/`
   - Firefox: 访问 `about:addons`

3. **启用开发者模式**
   - Chrome/Edge: 右上角开启"开发者模式"
   - Firefox: 无需开发者模式

4. **加载插件**
   - Chrome/Edge: 点击"加载已解压的扩展程序"
   - Firefox: 点击"从文件安装附加组件"
   - 选择 `docker_github_fixer` 文件夹

5. **确认安装**
   - 浏览器会提示安装确认
   - 点击"添加"或"确认"

### 方法二：从应用商店安装（未来计划）

1. **访问应用商店**
   - Chrome: Chrome Web Store
   - Edge: Microsoft Edge Add-ons
   - Firefox: Firefox Add-ons

2. **搜索插件**
   - 搜索 "Docker & GitHub 急速访问修复器"

3. **点击安装**
   - 按照提示完成安装

## 使用指南

### 基本使用

1. **启用插件**
   - 在扩展管理页面确保插件已启用
   - 插件图标会显示在浏览器工具栏

2. **访问目标网站**
   - 访问 `https://github.com` 或 `https://docker.com`
   - 插件会自动优化访问

3. **查看进度**
   - 页面顶部会显示蓝色进度条
   - 实时显示加载进度

### 高级设置

1. **打开设置面板**
   - 点击浏览器工具栏中的插件图标
   - 弹出设置面板

2. **调整优化策略**
   - **缓存优化**：启用/禁用资源缓存
   - **并行加载**：启用/禁用并行资源加载
   - **DNS优化**：启用/禁用DNS解析优化
   - **进度条**：启用/禁用访问进度条

3. **测试连接**
   - 点击"测试连接"按钮
   - 插件会测试GitHub和Docker的连接状态

4. **应用设置**
   - 调整设置后点击"应用设置"
   - 设置会立即生效

## 技术原理

### 网络优化技术

1. **请求拦截与处理**
   - 使用 `webRequest` API 拦截网络请求
   - 根据请求类型应用不同的优化策略
   - 智能选择最优的请求路径

2. **镜像源智能切换**
   - 内置多个GitHub和Docker镜像源
   - 自动检测镜像源可用性
   - 主站不可用时自动切换到备用源

3. **DNS优化**
   - 预解析常用域名
   - 缓存DNS解析结果
   - 优先选择响应速度快的DNS服务器

4. **连接优化**
   - 复用TCP连接
   - 优化SSL/TLS握手
   - 减少连接建立时间

### 性能优化技术

1. **资源管理**
   - 智能缓存静态资源
   - 压缩资源传输
   - 优先加载关键资源

2. **并行处理**
   - 同时加载多个资源
   - 优化资源加载顺序
   - 减少页面阻塞时间

3. **预测算法**
   - 基于历史数据预测加载时间
   - 智能调整优化策略
   - 自适应网络环境

## 故障排除

### 常见问题

1. **插件不生效**
   - 检查插件是否已启用
   - 刷新页面后重试
   - 检查浏览器版本是否支持

2. **GitHub仍然无法访问**
   - 点击插件图标测试连接状态
   - 尝试切换到不同的镜像源
   - 检查系统防火墙设置

3. **Docker镜像拉取慢**
   - 确保镜像加速源已配置
   - 检查网络连接稳定性
   - 尝试重启Docker服务

4. **进度条不显示**
   - 检查设置中是否启用了进度条
   - 刷新页面后重试
   - 检查浏览器是否支持相关API

### 日志查看

1. **浏览器控制台**
   - 按 `F12` 打开开发者工具
   - 切换到"控制台"标签
   - 查看插件输出的日志信息

2. **插件状态**
   - 点击插件图标
   - 查看当前状态信息
   - 检查连接测试结果

## 安全说明

### 权限说明

插件需要以下权限：

1. **webRequest**：拦截和优化网络请求
2. **storage**：存储用户设置和缓存数据
3. **tabs**：监控标签页状态
4. **scripting**：注入内容脚本

### 隐私保护

- **无数据收集**：插件不会收集任何个人信息
- **本地存储**：所有数据存储在本地
- **透明操作**：所有优化操作对用户可见
- **安全连接**：只处理安全的HTTPS连接
