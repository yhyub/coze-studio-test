# Docker & GitHub 急速访问修复器插件

## 插件功能说明

这是一个专门为修复Docker和GitHub访问问题而设计的浏览器插件，具有以下核心功能：

### 1. Docker访问修复
- **镜像加速源配置**：自动配置多个国内镜像加速源
- **Docker Hub访问优化**：解决访问超时和连接重置问题
- **API版本兼容性**：自动适配不同Docker API版本

### 2. GitHub访问修复
- **hosts文件优化**：自动添加最优GitHub IP地址
- **DNS解析加速**：优先选择最快的DNS服务器
- **镜像源切换**：智能切换到可用的GitHub镜像源
- **资源加载优化**：并行加载静态资源

### 3. 访问速度优化
- **资源缓存**：智能缓存常用页面资源
- **连接复用**：减少建立连接的时间
- **并行加载**：同时加载多个资源
- **压缩优化**：优先使用压缩资源

### 4. 实时进度条
- **访问进度可视化**：实时显示页面加载进度
- **平滑动画效果**：流畅的进度条动画
- **智能预测**：基于历史数据预测加载时间

## 安装方法

1. **下载插件包**：将本文件保存为 `docker_github_fixer.zip`
2. **解压**：解压到本地文件夹
3. **安装到浏览器**：
   - 打开浏览器扩展管理页面
   - 开启"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择解压后的文件夹

## 使用方法

1. **启用插件**：在扩展管理页面启用插件
2. **访问Docker或GitHub**：插件会自动优化访问
3. **查看状态**：点击插件图标查看当前状态
4. **调整设置**：在插件弹出页面调整优化策略

## 技术原理

### 网络请求优化
- **请求拦截**：使用 `webRequest` API 拦截网络请求
- **智能路由**：根据网络状况选择最优路径
- **头部优化**：优化HTTP请求头部，减少传输量

### 速度提升技术
- **DNS预解析**：提前解析域名
- **资源预加载**：预加载可能需要的资源
- **连接池管理**：复用TCP连接
- **压缩传输**：优先使用gzip压缩

### 错误处理
- **自动重试**：网络错误时自动重试
- **镜像源切换**：主站不可用时切换到镜像源
- **超时处理**：智能处理请求超时

## 系统要求

- **浏览器**：Chrome 90+ 或 Edge 90+
- **操作系统**：Windows、macOS、Linux
- **网络**：稳定的互联网连接

## 注意事项

1. **权限说明**：插件需要网络请求权限来优化访问
2. **隐私保护**：插件不会收集任何个人信息
3. **性能影响**：插件设计为轻量级，不会影响浏览器性能
4. **兼容性**：与其他扩展兼容，不会产生冲突

## 故障排除

### 常见问题

1. **插件不生效**
   - 检查是否启用了插件
   - 刷新页面后重试
   - 检查浏览器版本是否符合要求

2. **访问速度没有提升**
   - 检查网络连接是否稳定
   - 尝试清除浏览器缓存
   - 调整插件设置中的优化策略

3. **GitHub仍然无法访问**
   - 检查网络防火墙设置
   - 尝试切换到不同的镜像源
   - 检查系统hosts文件是否正确

4. **Docker命令失败**
   - 检查Docker服务是否正常运行
   - 验证镜像加速配置是否生效
   - 尝试重启Docker服务

## 配置说明

### 插件设置选项

1. **优化级别**
   - 基础：仅启用核心优化
   - 标准：启用大部分优化功能
   - 高级：启用所有优化功能

2. **镜像源选择**
   - 自动：根据网络状况自动选择
   - 手动：手动指定镜像源

3. **进度条设置**
   - 启用/禁用进度条
   - 调整进度条样式
   - 设置动画速度

4. **缓存设置**
   - 缓存大小限制
   - 缓存过期时间
   - 缓存清理策略
